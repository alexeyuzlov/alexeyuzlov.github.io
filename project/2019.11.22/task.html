Проект

Сделать динамическую форму квиза для паблика с генерацией контролов. Другими словами переносим консольный квиз полностью в браузер. Использовать за основу пример с последнего занятия (в папке examples)

Задание от 2019.11.22

  • теория по формам
  • в зависимости от type вопроса должен генерироваться динамический контрол .js-form-control:
    • type=radio, для одного варианта ответа. На выходе должно быть значение из data-value
    • type=checkbox, для нескольких вариантов ответа. На выходе массив из data-value. Этот случай уже реализован в examples
    • select, для одного варианта ответа. Этот контрол является альтернативным к радио и удобнее в использовании, когда предлагается много вариантов ответа
    • (опционально) type=text, для пользовательского ввода. Т.к. это пользовательский ввод предусмотрите дополнительную валидацию
  • для каждого вопроса динамический контрол .js-form-control должен генерится заново
  • форма должна иметь простейшую валидацию (хотя бы не принимать пустой ответ). Т.е. при попытке отправить форму создавать .error с сообщением о невалидности.
  • квиз должен заканчиваться статистикой по прохождению, т.е. обычный html виджет на ваше усмотрение.
  • (опционально) Можно добавить функционал таймера и через setInterval обновлять DOM элемент таймера. Соответственно по setTimeout квиз прерывается и показывает статистику прохождения
  • (опционально) Реализовать фичу кнопки назад, которая возвращает к предыдущему вопросу. При этом форма должна быть инициализирована с последним ответом пользователя.
